Security & Access Controls

Operational security model for VISIVA® authentication, route protection, and governance auditability.

Authentication Model

Credential Validation

Username/email and password are verified against a hashed credential store in SQLite.

Session Token

JWT is issued on success and stored in an HttpOnly cookie for protected route access.

Failure Controls

Rate limiting and lockout thresholds reduce brute-force and credential stuffing risk.

Audit & Governance

Login Attempt Ledger

Successful and failed attempts are recorded with identifier and source IP metadata.

Security Events

Login success/failure, temporary lockouts, and logout actions are captured in audit events.

Admin Visibility

Authenticated admin users can access recent audit records through protected API endpoints.

Operational Checklist

Rotate Secrets

Set and periodically rotate `JWT_SECRET` in production environments.

HTTPS Enforcement

Use TLS termination and secure cookie transport in deployed infrastructure.

Retention Policy

Archive and rotate authentication logs according to governance requirements.
